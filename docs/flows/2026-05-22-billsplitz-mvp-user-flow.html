BillSplitz MVP

High-Fidelity User Flow

A single-device iPhone workflow for moving from a receipt to a shareable settlement. The flow prioritizes manual correction, clear split ownership, and local recovery over automation.

Key Decisions

Manual Is First-Class

OCR accelerates entry but never owns the workflow. Users must be able to finish a split by typing items manually.

One Active Session

The MVP can persist one current split well. A session list/history can wait until the core loop is stable.

Share Text Before Image

A reliable text summary is more important than a polished share card. The card is a stretch layer.

3. Capture Receipt

The user can scan with VisionKit, import a receipt image, or bypass OCR. The bypass keeps the MVP useful even before OCR is reliable.

Services
  • ReceiptScanService wraps document camera.
  • PhotosUI handles import fallback.
  • Receipt image reference is stored locally.
Fallbacks
  • Denied camera: show import and manual entry.
  • No receipt photo: continue to manual review.
  • Scanner cancelled: return without losing session.

The receipt review screen is the control point for OCR errors. Users can edit line names, prices, quantities, categories, tax, tip, fees, and discounts before splitting.

  • Change category for preset behavior.
  • Delete noise lines from OCR.
  • Add missed items manually.
Data Output
  • Clean ReceiptItem records.
  • Session subtotal, tax, and tip.
  • Review status moves to splitting.

5. Split Board

The board shows each cleaned item and its current allocation. Presets prefill common cases, but every item remains overrideable.

Interaction Model
  • Assigned: one participant gets the full item.
  • Split: selected participants get equal shares.
  • Shared: all participants get equal shares.
  • Badges show recipients: blue means assigned, green means included in a shared split, and gray means available but not selected.
Implementation Notes
  • SplitRuleEngine creates default assignments.
  • ItemAssignment.shareRatio stores split weights.
  • Block settlement while required items are unassigned.

6. Settlement

The settlement screen translates assignments into per-person totals. It must make the math legible enough that users trust the result before sharing.

Calculator Rules
  • Round item shares, tax, and tip to cents.
  • Allocate any remainder to the largest pre-rounded share.
  • Fail if required items have no assignment.
Expected Tests
  • Existing SettlementCalculator tests stay green.
  • Add coverage for service fees and discounts.
  • Smoke test happy path through settlement.

7. Share Output

The user sends a clear summary through any messaging app. Plain text is the MVP requirement; a visual card is a polish layer if time remains.

Plain Text Format
  • Session title and payer.
  • Each participant's total.
  • Optional payment handle.
  • Receipt total reconciliation note.
Out of Scope
  • No Venmo/Zelle/Cash App API requests.
  • No accounts or shared editing.
  • No cloud receipt storage.

Error and Recovery Paths

Camera denied

Show photo import and manual entry. The app remains usable without camera permission.

OCR weak

Send raw lines to review with low-confidence labels. Never skip the review step.

Missing item

User adds a manual receipt line in review. It behaves like an OCR-derived item after save.

Unassigned item

Settlement button shows which items still need assignment and returns user to the split board.

App closed

SwiftData reloads the active session, participants, items, and assignments on next launch.

Implementation Mapping

View Models
  • SessionSetupViewModel: title, participants, preset.
  • ReceiptCaptureViewModel: scan/import/manual routing.
  • ReceiptReviewViewModel: editable items, subtotal, tax, tip.
  • SplitBoardViewModel: assignment state and preset overrides.
  • SettlementViewModel: totals and share output.
Services
  • SessionRepository: SwiftData persistence.
  • ReceiptOCRService: Vision text recognition.
  • ReceiptParserService: raw text to editable candidate items.
  • SplitRuleEngine: default assignments.
  • ShareExportService: text and optional image card.
